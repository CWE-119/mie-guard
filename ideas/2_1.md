## 1. Introduction & Threat Model

The Intel Management Engine (ME) and AMD Platform Security Processor (PSP) are separate microcontrollers embedded in the Platform Controller Hub (PCH) or CPU die. They operate **completely outside the main OS** with full access to system memory, network interfaces, and peripherals.

A compromised ME/PSP can:
- Read/write any RAM region via DMA (bypassing IOMMU).
- Send/receive network traffic through a **dedicated MAC address** invisible to the OS.
- Persist across reboots and OS reinstalls.
- Remain undetectable by any software running on the main CPU.

This document provides a complete blueprint for constructing a **hardware‑based monitoring and mitigation platform** that operates **outside the trust boundary** of the target PC. It fuses multiple independent sensors into a single, deterministic real‑time system.

## 3. The Seven Senses: Monitoring Vectors

To guarantee **zero blind spots** and **lossless telemetry**, the Sentry employs seven independent hardware taps.

| # | Sense | Physical Tap | FPGA Core / Processing | Detection Guarantee |
|---|-------|--------------|------------------------|---------------------|
| 1 | **Network Egress** | Passive Ethernet tap (inline RJ45) | MAC Frame Sniffer with IEEE 1588 timestamping | Every packet from PCH MAC is duplicated losslessly up to line rate. |
| 2 | **PCIe/DMA Traffic** | PCIe slot interposer (Gen3/4) | PCIe Protocol Analyzer (sniffs TLPs) | Zero‑copy capture of all TLPs; detects PCH‑initiated DMA reads. |
| 3 | **SPI Flash Changes** | SOIC‑8 clip on BIOS chip | SPI Slave Sniffer (snoops MOSI/MISO) | Immediate alert if Write Enable latch is set outside of boot. |
| 4 | **ME Debug Console** | LPC header (LAD0‑3, LFRAME) | LPC UART Decoder (115200 baud) | Real‑time capture of ME `printf` output. |
| 5 | **Power Signature** | Hall effect sensor (ACS712) on 3.3VSB | CIC filter + DMA streaming ADC samples | Detects ME wake‑ups even in S5 (soft‑off). |
| 6 | **JTAG/DCI (Optional)** | USB‑C DCI cable or JTAG header | OpenOCD bridge on ARM | Active intervention: halt ME core, set breakpoints. |
| 7 | **Physical Presence** | Reed switch + ambient light sensor | Tamper detection on Sentry enclosure | Ensures Sentry itself is not physically compromised. |

## 4. Real-Time Processing Pipeline

The FPGA fabric guarantees **deterministic, lossless processing** at hardware speeds.

### 4.1 Data Flow

1. **Ingestion:** Seven parallel DMA engines write incoming byte streams into Block RAM (BRAM) circular buffers.
2. **Pre‑Filtering:** Hardware pattern matching using Content‑Addressable Memory (CAM).
   - *Example:* If Ethernet packet source MAC == `00:15:17:xx:xx:xx` AND destination IP is not on LAN subnet → **Raise Interrupt**.
   - *Example:* If SPI command == `0x06` (Write Enable) and address ≥ `ME_Region_Base` → **Raise Interrupt**.
3. **Timestamping:** A central GPS‑Disciplined Oscillator (GPSDO) provides a 100 MHz clock with <50 ns accuracy. Every event is stamped with a 64‑bit TAI (International Atomic Time) value.
4. **Lossless Aggregation:** The FPGA writes a **Unified Event Stream** to a DDR4 buffer shared with the ARM cluster. The buffer is **triple‑buffered**; the ARM reads one buffer while the FPGA fills another. **No packets are ever dropped due to OS scheduling.**

### 4.2 ARM Software Stack

| Core | OS / Runtime | Responsibility |
|------|--------------|----------------|
| **Core 0** | FreeRTOS (bare‑metal) | **Alert Actuator:** on interrupt, cuts power to LAN PHY within **10 µs**. Hardware guarantee, not software. |
| **Cores 1‑3** | Linux with `PREEMPT_RT` | • Suricata IDS in `AF_PACKET` mode (duplicated Ethernet stream).<br>• TensorFlow Lite Micro running power side‑channel ML model.<br>• Web GUI (management port only). |

## 5. Phase 1: The Network Gatekeeper (Lossless Packet Capture)

**Goal:** Detect any network communication from the ME's dedicated MAC address.

### 5.1 Hardware Components

| Item | Model | Approx. Cost |
|------|-------|--------------|
| SBC | NVIDIA Jetson Orin Nano Developer Kit | $199 |
| Dual‑port NIC | Intel I350‑T2 (PCIe x4) | $40 |
| M.2 to PCIe adapter | For connecting NIC to Jetson | $15 |
| Power supply | 5V/4A USB‑C | $10 |
| Ethernet cables | 2× CAT6 | $5 |

### 5.2 Step‑by‑Step Assembly

1. **Assemble Jetson Orin Nano** according to NVIDIA's quick start guide.
2. **Connect the Intel I350‑T2 NIC** to the M.2 Key M slot via the adapter. This provides two physical ports: `eth0` and `eth1`.
3. **Install Ubuntu 22.04 LTS** (or JetPack 6.0) and update all packages.
4. **Disable IP configuration** on both interfaces:
   ```bash
   sudo nmcli device set eth0 managed no
   sudo nmcli device set eth1 managed no
   ```
5. **Create a transparent bridge**:
   ```bash
   sudo ip link add name br0 type bridge
   sudo ip link set dev eth0 master br0
   sudo ip link set dev eth1 master br0
   sudo ip link set up br0
   ```
6. **Install Suricata**:
   ```bash
   sudo apt install suricata
   ```
7. **Configure Suricata in AF_PACKET IPS mode** (edit `/etc/suricata/suricata.yaml`):
   ```yaml
   af-packet:
     - interface: br0
       cluster-id: 99
       cluster-type: cluster_flow
       defrag: yes
       use-mmap: yes
   ```
8. **Add custom rules** (`/etc/suricata/rules/mei.rules`):
   ```
   alert ether any any -> any any (msg:"Intel ME MAC active"; eth.src:00:15:17; sid:1000001; rev:1;)
   alert ether any any -> any any (msg:"Intel ME MAC active"; eth.src:3C:EC:EF; sid:1000002; rev:1;)
   ```
9. **Start Suricata**:
   ```bash
   sudo suricata -c /etc/suricata/suricata.yaml -i br0 --af-packet
   ```

**Result:** Any packet from an Intel ME OUI triggers an immediate alert logged to `eve.json`.

## 6. Phase 2: The SPI Guard (Firmware Tampering Detection)

**Goal:** Detect unauthorised writes to the SPI flash chip containing the ME firmware.

### 6.1 Hardware Components

| Item | Model | Approx. Cost |
|------|-------|--------------|
| USB‑to‑SPI bridge | FTDI FT232H breakout (Adafruit 2264) | $15 |
| SOIC‑8 test clip | Pomona 5250 | $15 |
| Jumper wires | Female‑to‑female Dupont | $5 |

### 6.2 Wiring (Passive Sniffing)

| FT232H Pin | SOIC‑8 Clip Pin | Signal |
|------------|-----------------|--------|
| `ADBUS0` (SCK) | Pin 6 | CLK |
| `ADBUS1` (MOSI) | Pin 5 | SI |
| `ADBUS2` (MISO) | Pin 2 | SO |
| `ADBUS3` (CS) | Pin 1 | CS# |
| `GND` | Pin 4 | Ground |
| **Do NOT connect 3.3V** | Pin 8 | VCC (powered by motherboard) |

### 6.3 Software Setup

1. **Install `flashrom`** on the Sentry (or a separate laptop):
   ```bash
   sudo apt install flashrom
   ```
2. **Load the FTDI kernel module**:
   ```bash
   sudo modprobe ftdi_sio
   echo 0403 6014 | sudo tee /sys/bus/usb-serial/drivers/ftdi_sio/new_id
   ```
3. **Create a baseline hash** of the ME firmware region:
   ```bash
   sudo flashrom -p ft2232_spi:type=232H -r backup.bin
   sha256sum backup.bin > baseline.sha256
   ```
4. **Monitoring script (`spi_guard.py`)** – runs continuously, checking the SPI Status Register:
   ```python
   import time, subprocess, hashlib

   def read_status():
       out = subprocess.check_output(["flashrom", "-p", "ft2232_spi:type=232H",
                                      "-c", "MX25L12835F", "-r", "/dev/stdout",
                                      "-i", "0:1"])
       return out[0]

   while True:
       sr = read_status()
       if sr & 0x02:   # Write Enable Latch (WEL) set
           print("CRITICAL: SPI Write Enable Latch set outside of boot!")
           # Send alert via syslog or webhook
       time.sleep(1)
   ```

**Result:** Any attempt to modify the ME firmware triggers a real‑time alert.

## 7. Phase 3: The Power Oracle (Stealth Activity Detection)

**Goal:** Detect ME activity via its power consumption signature, even when network and SPI are silent.

### 7.1 Hardware Components

| Item | Model | Approx. Cost |
|------|-------|--------------|
| USB oscilloscope | Digilent Analog Discovery 3 | $300 |
| Current & power adapter | Digilent Current & Power Adapter | $70 |

### 7.2 Connection

- **Non‑invasive:** Clamp the current probe around the **purple +5VSB wire** of the ATX 24‑pin connector.
- **More sensitive:** Identify the **3.3VSB pin** on the LPC debug header (often pin 8 or 9) and connect the adapter in‑line.

### 7.3 Software Setup

1. **Install WaveForms SDK** or `pydwf` on the Sentry ARM Linux.
2. **Capture baseline traces**:
   - PC off (S5) → flat current draw.
   - PC on, AMT ping active → periodic 2‑4 Hz "blips".
3. **Deploy detection script** using SciPy FFT to identify the ME's 32 kHz clock harmonic:
   ```python
   import numpy as np
   from scipy import signal

   def detect_me(trace, fs):
       f, Pxx = signal.periodogram(trace, fs)
       peak_32k = np.max(Pxx[(f > 31000) & (f < 33000)])
       baseline = np.median(Pxx)
       return peak_32k > baseline * 10
   ```
4. **Integrate with Sentry:** Stream ADC data via DMA to the ARM core running TensorFlow Lite Micro for real‑time inference.

**Result:** Even a silent ME that never touches the network or SPI is detectable.

## 8. Phase 4: The PCIe Arbiter (Ultimate Control)

**Goal:** Actively block malicious DMA reads initiated by the PCH/ME.

### 8.1 Hardware Components

| Item | Model | Approx. Cost |
|------|-------|--------------|
| M.2 FPGA board | PicoEVB (Lattice ECP5) or NiteFury (Artix‑7) | $50–$200 |
| M.2 to PCIe adapter | For external connection | $10 |
| JTAG programmer | FT2232H Mini Module | $30 |

### 8.2 Implementation Overview

1. **Install FPGA board** in an M.2 slot (or use riser).
2. **Develop Verilog/VHDL** using open‑source PCIe frameworks (e.g., RIFFA, LiteX).
3. **Program the FPGA** to:
   - Sniff all **Memory Read Request (MRd)** TLPs.
   - Check **Requester ID** against known PCH BDFs:
     - Intel: `00:1f.x`, `00:16.x`, `00:14.x`
     - AMD: `00:14.x`, `00:08.x`
   - If a match is found and address is outside whitelisted ranges → **drop packet** and raise alert.

**Example Verilog snippet:**
```verilog
always @(posedge clk) begin
    if (tlp_valid && tlp_type == MRd && requester_id inside {16'h001f, 16'h0016}) begin
        if (address[63:32] != 0 || address[31:0] > 32'h10000000) begin
            drop_packet <= 1;
            alert       <= 1;
        end
    end
end
```

**Result:** The ME's ability to read arbitrary system memory via DMA is neutered.

## 9. The Ultimate Killswitch: Halting the ME

For compatible hardware, you can permanently disable most ME functionality using `me_cleaner`.

**⚠️ WARNING: This process can permanently brick your motherboard. Proceed only on a system you can afford to lose.**

1. **Dump the full BIOS ROM** using the SPI hardware from Phase 2:
   ```bash
   sudo flashrom -p ft2232_spi:type=232H -r original.bin
   ```
2. **Run `me_cleaner`** on a trusted Linux machine:
   ```bash
   python me_cleaner.py -s original.bin   # -s soft‑disable (reversible)
   # OR
   python me_cleaner.py -S original.bin   # -S hard‑disable (irreversible)
   ```
3. **Reflash the modified firmware**:
   ```bash
   sudo flashrom -p ft2232_spi:type=232H -w modified.bin
   ```

## 10. Integration with `mei-guard` Software

All hardware phases feed telemetry into the **`mei-guard`** software stack (C daemon + Python analyser).

- **Phase 1 (Suricata):** Forward alerts via syslog to the analyser.
- **Phase 2 (SPI Guard):** `spi_guard.py` writes JSON alerts to a Unix socket.
- **Phase 3 (Power Oracle):** Python script sends HTTP POSTs to the analyser's webhook endpoint.
- **Phase 4 (PCIe Arbiter):** FPGA exposes a UART/I²C interface; a small daemon translates interrupts into `mei-guard` alerts.

The unified web dashboard (`http://<sentry-ip>:7473`) displays all alerts with nanosecond‑precision timestamps.

## 13. Limitations and Future Improvements

**Current Limitations:**
- Phase 3 requires per‑motherboard calibration.
- Phase 4 may reduce NVMe performance by 5–10%.
- Assumes the attacker has not physically tampered with the Sentry itself.

**Future Enhancements:**
- **Integrated PCB:** Combine RP2040/STM32 for SPI sniffing and a small FPGA on a single board managed by a CM4.
- **TinyML on Power Traces:** Deploy quantised models on the ARM for real‑time classification.
- **Automatic ME Halting:** Use HECI command `HMRFPO` over LPC to disable ME on alert.
